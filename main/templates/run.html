From the root directory you can excecute the run.py script with one or more of these arguments:

Flag Name Description
-w --watch Watch files for changes when running the development web server
-c --clean Recompiles files when running the development web server
-C --clean-all Cleans all the pip, Node & Bower related tools / libraries and updates them to their latest versions
-m --minify Compiles files into minified version before deploying
-s --start Starts the dev_appserver.py with datastore and blobstore paths
-o --host The host to start the dev_appserver.py
-p --port The port to start the dev_appserver.py
-f --flush Clears the datastore, blobstore, etc
--appserver-args ... Used with -s, all following args are passed to dev_appserver.py
gae-init-babel only
-u --pybabel-update Update existing message catalogs from messages.pot (pybabel update..)
-l --pybabel-init Create new message catalogs from messages.pot (pybabel init..)
-i --pybabel-init-missing Create new message catalogs from messages.pot that are defined in config.py and still not present (pybabel init..)
-b --pybabel-compile Compile message catalogs to MO files (pybabel compile..)
